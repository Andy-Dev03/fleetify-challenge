# Employee Attendance System – Frontend

A modern web-based frontend application for managing **Employees, Departments, and Attendance Logs**.  
This system is designed for enterprise-level HR operations, enabling companies to manage workforce discipline with accuracy and ease.

---

## Tech Stack

- **Next.js 15 (App Router)** – React framework for production-ready web apps
- **React 19**
- **TypeScript 5**
- **Tailwind CSS 4** – Utility-first CSS framework
- **React Toastify** – Elegant toast notifications
- **Lucide React** – Icon library

---

## Quick Start

### Prerequisites

- Node.js `>=18.18` or `>=20`
- npm / pnpm / yarn

### Install dependencies

```bash
npm install
```

### Run

```bash
# Development mode
npm run dev
```

## Features

### 1. CRUD Department

- **Create**  
  `POST /departement`

  ```json
  {
    "DepartmentName": "Engineering",
    "MaxClockInTimeStr": "08:30",
    "MaxClockOutTimeStr": "17:30"
  }
  ```

- **Read**

  - `GET /departements`
  - `GET /departement/:id`

- **Update**  
  `PATCH /departement/:id`

  ```json
  {
    "DepartmentName": "Engineering",
    "MaxClockInTimeStr": "08:30",
    "MaxClockOutTimeStr": "17:30"
  }
  ```

- **Delete**  
  `DELETE /departement/:id`

---

### 2. CRUD Employee

- **Create**  
  `POST /employee`

  > ⚠️ `EmployeeID` is auto-generated by backend (e.g., `EMP-001`) and must not be sent from frontend.

  ```json
  {
    "DepartmentID": 1,
    "Name": "Jane Doe",
    "Address": "Jakarta"
  }
  ```

- **Read**

  - `GET /employees`
  - `GET /employee/:id`

- **Update**  
  `PATCH /employee/:id`

  ```json
  {
    "DepartmentID": 2,
    "Name": "Jane Doe",
    "Address": "Bandung"
  }
  ```

- **Delete**  
  `DELETE /employee/:id`

---

### 3. Attendance Management

- **Check-in (Absen Masuk)**  
  `POST /attendance`

  > ⚠️ `AttendanceID` is auto-generated by backend (e.g., `ATT-001`) and must not be sent from frontend.

  ```json
  {
    "employee_id": "EMP-001",
    "clock_in": "2025-08-17 08:00:00"
  }
  ```

- **Check-out (Absen Keluar)**  
  `PUT /attendance/:attendance_id`
  ```json
  {
    "attendanceId": "ATT-001",
    "clock_out": "2025-08-17 17:00:00"
  }
  ```

---

### 4. Attendance Logs & Punctuality Analysis

- **Get Logs**  
  `GET /attendance/logs`  
  Supports filters:

  - `date=YYYY-MM-DD`
  - `department_id=1`

  Example:  
  `GET /attendance/logs?date=2025-08-17&department_id=2`

- **Response Example**

  ```json
  {
    "data": [
      {
        "attendance_id": "ATT-001",
        "attendance_type": 1,
        "clock_in": "08:00:00",
        "clock_out": "05:00:00",
        "date_attendance": "2025-08-17 07:58:00",
        "department": "Fleetify",
        "description": "On Time",
        "employee_id": "EMP-001",
        "id": 1,
        "name": "Andy"
      }
    ]
  }
  ```

- **Rules for Analysis**
  - **On Time (Check-in)** → if `clock_in <= Department.MaxClockInTime`
  - **Late (Check-in)** → if `clock_in > Department.MaxClockInTime`
  - **On Time (Check-out)** → if `clock_out >= Department.MaxClockOutTime`
  - **Early Leave** → if `clock_out < Department.MaxClockOutTime`
